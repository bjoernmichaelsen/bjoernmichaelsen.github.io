LibreOffice EasyHacks Rotation

I just rotated the featured items on the LibreOffice EasyHacks Page. If you are interested in contributing and wondered what you could do, here are a few selected suggestions:

  • Improving EasyHacks Bugzilla-Wiki integration (Skills: PHP)
  • find / fix all German spellings in internal APIs (Skills: grep, German, code reading)
  • strip out non-trivial globals before main (Skills: build, simple C++)
  • Improve build with/without symbols (Skills: build, autoconf, Perl)
  • warning-free code (Skills: building the beast, basic C++ skills and the attitude to ask when in doubt)
  • Cleanup extensions list (Skills: C++, common sense, build)
  • Make macro editing easier (Skills: simple C++ programming, willingness to learn a bit of VCL)
  • audit / remove SvStream long operators (Skills: building, simple C++ programming)
  • calc sheet tab widget screen space usage
  • Change visibility of comments / notes via Notes Ruler Control (Skills: C++)
  • default to SQLite not HSQLDB in Base (Skills: build, C++)
You will find more info on the LibreOffice EasyHacks Page and an even wider range of things to do on the EasyHacks by required Skill Page.
